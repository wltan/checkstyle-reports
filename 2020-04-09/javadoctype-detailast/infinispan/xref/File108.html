45     /**
46      * A method that adds flags to any API call.  For example, consider the following code snippet:
47      * <pre>
48      *   cache.withFlags(Flag.FORCE_WRITE_LOCK).get(key);
49      * </pre>
50      * will invoke a cache.get() with a write lock forced.
51      * <p />
52      * <b>Note</b> that for the flag to take effect, the cache operation <b>must</b> be invoked on the instance returned by
53      * this method.
54      * <p />
55      * As an alternative to setting this on every
56      * invocation, users could also consider using the {@link DecoratedCache} wrapper, as this allows for more readable
57      * code.  E.g.:
58      * <pre>
59      *    Cache forceWriteLockCache = new DecoratedCache(cache, Flag.FORCE_WRITE_LOCK);
60      *    forceWriteLockCache.get(key1);
61      *    forceWriteLockCache.get(key2);
62      *    forceWriteLockCache.get(key3);
63      * </pre>
64      *
65      * @param flags a set of flags to apply.  See the {@link Flag} documentation.
66      * @return an {@link AdvancedCache} instance on which a real operation is to be invoked, if the flags are
67      * to be applied.
68      */
69     AdvancedCache<K, V> withFlags(Flag... flags);

269    /**
270     * Using this operation, users can call any {@link AdvancedCache} operation
271     * with a given {@link ClassLoader}. This means that any {@link ClassLoader} happening
272     * as a result of the cache operation will be done using the {@link ClassLoader}
273     * given. For example:
274     * <p />
275     * When users store POJO instances in caches configured with {@link org.infinispan.configuration.cache.StoreAsBinaryConfiguration},
276     * these instances are transformed into byte arrays. When these entries are
277     * read from the cache, a lazy unmarshalling process happens where these byte
278     * arrays are transformed back into POJO instances. Using {@link AdvancedCache#with(ClassLoader)}
279     * when reading that enables users to provide the class loader that should
280     * be used when trying to locate the classes that are constructed as a result
281     * of the unmarshalling process.
282     * <pre>
283     *    cache.with(classLoader).get(key);
284     * </pre>
285     * <b>Note</b> that for the flag to take effect, the cache operation <b>must</b> be invoked on the instance
286     * returned by this method.
287     * <p />
288     * As an alternative to setting this on every
289     * invocation, users could also consider using the {@link DecoratedCache} wrapper, as this allows for more readable
290     * code.  E.g.:
291     * <pre>
292     *    Cache classLoaderSpecificCache = new DecoratedCache(cache, classLoader);
293     *    classLoaderSpecificCache.get(key1);
294     *    classLoaderSpecificCache.get(key2);
295     *    classLoaderSpecificCache.get(key3);
296     * </pre>
297     *
298     * @return an {@link AdvancedCache} instance upon which operations can be called
299     * with a particular {@link ClassLoader}.
300     */
301    AdvancedCache<K, V> with(ClassLoader classLoader);

414    /**
415     * Gets a collection of entries, returning them as {@link Map} of the values
416     * associated with the set of keys requested.
417     * <p>
418     * If the cache is configured read-through, and a get for a key would
419     * return null because an entry is missing from the cache, the Cache's
420     * {@link CacheLoader} is called in an attempt to load the entry. If an
421     * entry cannot be loaded for a given key, the returned Map will contain null for
422     * value of the key.
423     * <p>
424     * Unlike other bulk methods if this invoked in an existing transaction all entries
425     * will be stored in the current transactional context
426     * <p>
427     * The returned {@link Map} will be a copy and updates to the map will not be reflected
428     * in the Cache and vice versa.  The keys and values themselves however may not be
429     * copies depending on if storeAsBinary is enabled and the value was retrieved from
430     * the local node.
431     *
432     * @param keys The keys whose associated values are to be returned.
433     * @return A map of entries that were found for the given keys. If an entry is not
434     *         found for a given key, it will not be in the returned map.
435     * @throws NullPointerException  if keys is null or if keys contains a null
436     */
437    Map<K, V> getAll(Set<?> keys);

475    /**
476     * Retrieve the entry iterable that can be used to iterate over the contents of this cache.  Note that every
477     * invocation of {@link Iterable#iterator()} will cause a new request chain to retrieve all of the values in
478     * the cache, including any configured loaders.  This iterator is performed in memory sensitive way and thus works
479     * for distributed caches unlike {@link Cache#entrySet()}.
480     * <p>Whenever possible a user should provide a converter through
481     * {@link org.infinispan.iteration.EntryIterable#converter(org.infinispan.filter.Converter)} to reduce value
482     * payloads or create projection views.</p>
483     * <p>This iterator does not participate in any ongoing transaction that may be present.  This is because
484     * {@link org.infinispan.util.concurrent.IsolationLevel#REPEATABLE_READ} transactions have to keep track of all
485     * the entries they have read (which in this case would include the entries that did not match the filter).
486     * Keeping all the values on a single node would negate the benefits of using a filter and/or converter, and in many
487     * cases it would cause OutOfMemoryErrors.  If it is desired to remove entries the
488     * {@link org.infinispan.Cache#remove(Object)} method or another similar method may be used which would
489     * participate in the transaction</p>
490     * <p>The iterable should always be closed after finished with it, using the standard
491     * try/finally or try with resource idioms to ensure that any current resources are freed if an exception prevents
492     * full iteration of iterator.  Note this will prevent any ongoing iterators that were created from it from
493     * progressing further.</p>
494     * @param filter The filter to use.  Note this is required and for distributed caches must be serializable.  Callbacks
495     *               to the filter will never provide a key or value that will be null.
496     */
497    EntryIterable<K, V> filterEntries(KeyValueFilter<? super K, ? super V> filter);

499    /**
500     * It fetches all the keys which belong to the group.
501     * <p/>
502     * Semantically, it iterates over all the keys in memory and persistence, and performs a read operation in the keys
503     * found. Multiple invocations inside a transaction ensures that all the keys previous read are returned and it may
504     * return newly added keys to the group from other committed transactions (also known as phantom reads).
505     * <p/>
506     * The {@code map} returned is immutable and represents the group at the time of the invocation. If you want to add
507     * or remove keys from a group use {@link #put(Object, Object)} and {@link #remove(Object)}. To remove all the keys
508     * in the group use {@link #removeGroup(String)}.
509     * <p/>
510     * To improve performance you may use the {@code flag} {@link org.infinispan.context.Flag#SKIP_CACHE_LOAD} to avoid
511     * fetching the key/value from persistence. However, you will get an inconsistent snapshot of the group.
512     *
513     * @param groupName the group name.
514     * @return an immutable {@link java.util.Map} with the key/value pairs.
515     */
516    Map<K, V> getGroup(String groupName);
517 
518    /**
519     * It removes all the key which belongs to a group.
520     * <p/>
521     * Semantically, it fetches the most recent group keys/values and removes them.
522     * <p/>
523     * Note that, concurrent addition perform by other transactions/threads to the group may not be removed.
524     *
525     * @param groupName the group name.
526     */
527    void removeGroup(String groupName);
