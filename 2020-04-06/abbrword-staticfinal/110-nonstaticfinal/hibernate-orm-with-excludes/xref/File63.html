62   * <br>
63   * Compatible with Cach&eacute; 2007.1.

65   * <head>
66   * <title>Cach&eacute; and Hibernate</title>
67   * </head>
68   * <body>
69   * <h1>Cach&eacute; and Hibernate</h1>
70   * <h2>PREREQUISITES</h2>
71   * These setup instructions assume that both Cach&eacute; and Hibernate are installed and operational.
72   * <br>
73   * <h2>HIBERNATE DIRECTORIES AND FILES</h2>
74   * JBoss distributes the InterSystems Cache' dialect for Hibernate 3.2.1
75   * For earlier versions of Hibernate please contact
76   * <a href="http://www.intersystems.com/support/cache-support.html">InterSystems Worldwide Response Center</A> (WRC)
77   * for the appropriate source files.
78   * <br>
79   * <h2>CACH&Eacute; DOCUMENTATION</h2>
80   * Documentation for Cach&eacute; is available online when Cach&eacute; is running.
81   * It can also be obtained from the
82   * <a href="http://www.intersystems.com/cache/downloads/documentation.html">InterSystems</A> website.
83   * The book, "Object-oriented Application Development Using the Cach&eacute; Post-relational Database:
84   * is also available from Springer-Verlag.
85   * <br>
86   * <h2>HIBERNATE DOCUMENTATION</h2>
87   * Hibernate comes with extensive electronic documentation.
88   * In addition, several books on Hibernate are available from
89   * <a href="http://www.manning.com">Manning Publications Co</a>.
90   * Three available titles are "Hibernate Quickly", "Hibernate in Action", and "Java Persistence with Hibernate".
91   * <br>
92   * <h2>TO SET UP HIBERNATE FOR USE WITH CACH&Eacute;</h2>
93   * The following steps assume that the directory where Cach&eacute; was installed is C:\CacheSys.
94   * This is the default installation directory for  Cach&eacute;.
95   * The default installation directory for Hibernate is assumed to be C:\Hibernate.
96   * <p/>
97   * If either product is installed in a different location, the pathnames that follow should be modified appropriately.
98   * <p/>
99   * Cach&eacute; version 2007.1 and above is recommended for use with
100  * Hibernate.  The next step depends on the location of your
101  * CacheDB.jar depending on your version of Cach&eacute;.
102  * <ol>
103  * <li>Copy C:\CacheSys\dev\java\lib\JDK15\CacheDB.jar to C:\Hibernate\lib\CacheDB.jar.</li>
104  * <p/>
105  * <li>Insert the following files into your Java classpath:
106  * <p/>
107  * <ul>
108  * <li>All jar files in the directory C:\Hibernate\lib</li>
109  * <li>The directory (or directories) where hibernate.properties and/or hibernate.cfg.xml are kept.</li>
110  * </ul>
111  * </li>
112  * <p/>
113  * <li>In the file, hibernate.properties (or hibernate.cfg.xml),
114  * specify the Cach&eacute; dialect and the Cach&eacute; version URL settings.</li>
115  * </ol>
116  * <p/>
117  * For example, in Hibernate 3.2, typical entries in hibernate.properties would have the following
118  * "name=value" pairs:
119  * <p/>
120  * <table cols=3 border cellpadding=5 cellspacing=0>
121  * <tr>
122  * <th>Property Name</th>
123  * <th>Property Value</th>
124  * </tr>
125  * <tr>
126  * <td>hibernate.dialect</td>
127  * <td>org.hibernate.dialect.Cache71Dialect</td>
128  * </tr>
129  * <tr>
130  * <td>hibernate.connection.driver_class</td>
131  * <td>com.intersys.jdbc.CacheDriver</td>
132  * </tr>
133  * <tr>
134  * <td>hibernate.connection.username</td>
135  * <td>(see note 1)</td>
136  * </tr>
137  * <tr>
138  * <td>hibernate.connection.password</td>
139  * <td>(see note 1)</td>
140  * </tr>
141  * <tr>
142  * <td>hibernate.connection.url</td>
143  * <td>jdbc:Cache://127.0.0.1:1972/USER</td>
144  * </tr>
145  * </table>
146  * <p/>
147  * <dl>
148  * <dt><b>Note 1</b></dt>
149  * <dd>Please contact your administrator for the userid and password you should use when attempting access via JDBC.
150  * By default, these are chosen to be "_SYSTEM" and "SYS" respectively as noted in the SQL standard.</dd>
151  * </dl>
152  * <br>
153  * <h2>CACH&Eacute; VERSION URL</h2>
154  * This is the standard URL for the JDBC driver.
155  * For a JDBC driver on the machine hosting Cach&eacute;, use the IP "loopback" address, 127.0.0.1.
156  * For 1972, the default port, specify the super server port of your Cach&eacute; instance.
157  * For USER, substitute the NAMESPACE which contains your Cach&eacute; database data.
158  * <br>
159  * <h2>CACH&Eacute; DIALECTS</h2>
160  * Choices for Dialect are:
161  * <br>
162  * <p/>
163  * <ol>
164  * <li>org.hibernate.dialect.Cache71Dialect (requires Cach&eacute;
165  * 2007.1 or above)</li>
166  * <p/>
167  * </ol>
168  * <br>
169  * <h2>SUPPORT FOR IDENTITY COLUMNS</h2>
170  * Cach&eacute; 2007.1 or later supports identity columns.  For
171  * Hibernate to use identity columns, specify "native" as the
172  * generator.
173  * <br>
174  * <h2>SEQUENCE DIALECTS SUPPORT SEQUENCES</h2>
175  * <p/>
176  * To use Hibernate sequence support with Cach&eacute; in a namespace, you must FIRST load the following file into that namespace:
177  * <pre>
178  *     etc\CacheSequences.xml
179  * </pre>
180  * For example, at the COS terminal prompt in the namespace, run the
181  * following command:
182  * <p>
183  * d LoadFile^%apiOBJ("c:\hibernate\etc\CacheSequences.xml","ck")
184  * <p>
185  * In your Hibernate mapping you can specify sequence use.
186  * <p>
187  * For example, the following shows the use of a sequence generator in a Hibernate mapping:
188  * <pre>
189  *     &lt;id name="id" column="uid" type="long" unsaved-value="null"&gt;
190  *         &lt;generator class="sequence"/&gt;
191  *     &lt;/id&gt;
192  * </pre>
193  * <br>
194  * <p/>
195  * Some versions of Hibernate under some circumstances call
196  * getSelectSequenceNextValString() in the dialect.  If this happens
197  * you will receive the error message: new MappingException( "Dialect
198  * does not support sequences" ).
199  * <br>

404 	/**
405 	 * The syntax used to add a foreign key constraint to a table.
406 	 *
407 	 * @return String
408 	 */
409 	public String getAddForeignKeyConstraintString(
410 			String constraintName,
411 			String[] foreignKey,
412 			String referencedTable,
413 			String[] primaryKey,
414 			boolean referencesPrimaryKey) {
415 		// The syntax used to add a foreign key constraint to a table.
416 		return new StringBuilder( 300 )
417 				.append( " ADD CONSTRAINT " )
418 				.append( constraintName )
419 				.append( " FOREIGN KEY " )
420 				.append( constraintName )
421 				.append( " (" )
422 				.append( StringHelper.join( ", ", foreignKey ) )	// identifier-commalist
423 				.append( ") REFERENCES " )
424 				.append( referencedTable )
425 				.append( " (" )

561 	public LockingStrategy getLockingStrategy(Lockable lockable, LockMode lockMode) {
562 		// InterSystems Cache' does not current support "SELECT ... FOR UPDATE" syntax...
563 		// Set your transaction mode to READ_COMMITTED before using
564 		if ( lockMode==LockMode.PESSIMISTIC_FORCE_INCREMENT) {
565 			return new PessimisticForceIncrementLockingStrategy( lockable, lockMode);
566 		}
567 		else if ( lockMode==LockMode.PESSIMISTIC_WRITE) {
568 			return new PessimisticWriteUpdateLockingStrategy( lockable, lockMode);
569 		}
570 		else if ( lockMode==LockMode.PESSIMISTIC_READ) {
571 			return new PessimisticReadUpdateLockingStrategy( lockable, lockMode);
572 		}
573 		else if ( lockMode==LockMode.OPTIMISTIC) {
574 			return new OptimisticLockingStrategy( lockable, lockMode);
575 		}
576 		else if ( lockMode==LockMode.OPTIMISTIC_FORCE_INCREMENT) {
577 			return new OptimisticForceIncrementLockingStrategy( lockable, lockMode);
578 		}
579 		else if ( lockMode.greaterThan( LockMode.READ ) ) {
580 			return new UpdateLockingStrategy( lockable, lockMode );
581 		}
582 		else {
583 			return new SelectLockingStrategy( lockable, lockMode );
584 		}
585 	}

611 	public String getLimitString(String sql, boolean hasOffset) {
612 		if ( hasOffset ) {
613 			throw new UnsupportedOperationException( "query result offset is not supported" );
614 		}
615 
616 		// This does not support the Cache SQL 'DISTINCT BY (comma-list)' extensions,
617 		// but this extension is not supported through Hibernate anyway.
618 		int insertionPoint = sql.startsWith( "select distinct" ) ? 15 : 6;
619 
620 		return new StringBuilder( sql.length() + 8 )
621 				.append( sql )
622 				.insert( insertionPoint, " TOP ? " )
623 				.toString();
