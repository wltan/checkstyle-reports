38  /** Factory for predefined reference frames.
39   *
40   * <h1> FramesFactory Presentation </h1>
41   * <p>
42   * Several predefined reference {@link Frame frames} are implemented in OREKIT.
43   * They are linked together in a tree with the <i>Geocentric
44   * Celestial Reference Frame</i> (GCRF) as the root of the tree.
45   * This factory is designed to:
46   * </p>
47   * <ul>
48   *   <li>build the frames tree consistently,</li>
49   *   <li>avoid rebuilding some frames that may be costly to recreate all the time,</li>
50   *   <li>set up interpolation/caching features for some frames that may induce costly computation</li>
51   *   <li>streamline the {@link EOPHistory Earth Orientation Parameters} history loading.</li>
52   * </ul>
53   * <h2> Reference Frames </h2>
54   * <p>
55   * The user can retrieve those reference frames using various static methods, the most
56   * important ones being: {@link #getFrame(Predefined)}, {@link #getGCRF()},
57   * {@link #getCIRF(IERSConventions, boolean)} {@link #getTIRF(IERSConventions, boolean)},
58   * {@link #getITRF(IERSConventions, boolean)}, {@link #getEME2000()},
59   * {@link #getMOD(IERSConventions)}, {@link #getTOD(IERSConventions, boolean)},
60   * {@link #getGTOD(IERSConventions, boolean)}, {@link #getITRFEquinox(IERSConventions, boolean)},
61   * {@link #getTEME()} and {@link #getVeis1950()}.
62   * </p>
63   * <h2> International Terrestrial Reference Frame</h2>
64   * <p>
65   * This frame is the current (as of 2013) reference realization of
66   * the International Terrestrial Reference System produced by IERS.
67   * It is described in <a href="ftp://tai.bipm.org/iers/conv2010/tn36.pdf">
68   * IERS conventions (2010)</a>. It replaces the Earth Centered Earth Fixed
69   * frame which is the reference frame for GPS satellites.
70   * </p>
71   * <p>
72   * This frame is used to define position on solid Earth. It rotates with
73   * the Earth and includes the pole motion with respect to Earth crust as
74   * provided by {@link org.orekit.data.DataProvidersManager IERS data}.
75   * Its pole axis is the IERS Reference Pole (IRP).
76   * </p>
77   * <p>
78   * Previous realizations of the ITRS are available and linked together using
79   * {@link HelmertTransformation Helmert transformations}. Parameters for all
80   * ITRS realizations since 1988 are available from the ITRF site <a
81   * href="ftp://itrf.ensg.ign.fr/pub/itrf/ITRF.TP"> ftp://itrf.ensg.ign.fr/pub/itrf/ITRF.TP</a>).
82   * Orekit provides a {@link HelmertTransformation.Predefined#createTransformedITRF(Frame,
83   * String) utility} method to build simply several of them and link them together.
84   * </p>
85   * <p>
86   * ITRF can be built using the new non-rotating origin paradigm
87   * mandated by IAU 2000 resolution B1.8 and any supported {@link IERSConventions
88   * IERS conventions} (even IERS 1996 can be used with non-rotating origin paradigm,
89   * despite the resolution was not yet adopted at conventions publication time.
90   * </p>
91   * <p>
92   * ITRF can also be built using the classical equinox paradigm used prior to IAU 2000
93   * resolution B1.8 and any supported {@link IERSConventions IERS conventions} (even
94   * IERS 2003 and 2010 can be used with equinox paradigm, despite the resolution is
95   * in effect now). The choice of paradigm (non-rotating origin or equinox) and the
96   * choice of IERS conventions (i.e. the choice of precession/nutation models) can
97   * be made independently by user, Orekit provides all alternatives.
98   * </p>
99   * <h2>Intermediate frames</h2>
100  * <p>
101  * Orekit also provides all the intermediate frames that are needed to transform
102  * between GCRF and ITRF, along the two paths: ITRF/TIRF/CIRF/GCRF for the
103  * non-rotating origin paradigm and ITRF/GTOD/TOD/MOD/EME2000/GCRF for the equinox
104  * paradigm.
105  * </p>
106  * <h2> Earth Orientation Parameters </h2>
107  * <p>
108  * This factory also handles loading of Earth Orientation Parameters (EOP) needed
109  * for accurate transformations between inertial and Earth fixed frames, using
110  * {@link org.orekit.data.DataProvidersManager} features. EOP are IERS conventions
111  * dependent, because they correspond to correction to the precession/nutation
112  * models. When EOP should be applied, but EOP data are not available, then a null
113  * (0.0) correction is used. This can occur when no EOP data is loaded, or when the
114  * requested date is beyond the time span of the loaded EOP data. Using a null
115  * correction can result in coarse accuracy. To check the time span covered by EOP data use
116  * {@link #getEOPHistory(IERSConventions, boolean)}, {@link EOPHistory#getStartDate()},
117  * and {@link EOPHistory#getEndDate()}.
118  * <p>
119  * For more on configuring the EOP data Orekit uses see
120  * <a href="https://www.orekit.org/forge/projects/orekit/wiki/Configuration">
121  * https://www.orekit.org/forge/projects/orekit/wiki/Configuration</a>.
122  * <p>
123  * Here is a schematic representation of the predefined reference frames tree:
124  * </p>
125  * <pre>
126  *                                                                  GCRF
127  *                                                                    |
128  *                                                 |-----------------------------------------------
129  *                                                 |                         |     Frame bias     |
130  *                                                 |                         |                 EME2000
131  *                                                 |                         |                    |
132  *                                                 |                         | Precession effects |
133  *                                                 |                         |                    |
134  *           Bias, Precession and Nutation effects |                        MOD                  MOD  (Mean Equator Of Date)
135  *                                                 |                         |             w/o EOP corrections
136  *                                                 |                         |  Nutation effects  |
137  *    (Celestial Intermediate Reference Frame)   CIRF                        |                    |
138  *                                                 |                        TOD                  TOD  (True Equator Of Date)
139  *                          Earth natural rotation |                         |             w/o EOP corrections
140  *                                                 |-------------            |    Sidereal Time   |
141  *                                                 |            |            |                    |
142  *  (Terrestrial Intermediate Reference Frame)   TIRF         TIRF         GTOD                 GTOD  (Greenwich True Of Date)
143  *                                                 |    w/o tidal effects                  w/o EOP corrections
144  *                                     Pole motion |            |                                 |
145  *                                                 |            |                                 |-------------
146  *                                                 |            |                                 |            |
147  * (International Terrestrial Reference Frame)   ITRF         ITRF                              ITRF        VEIS1950
148  *                                                 |    w/o tidal effects                   equinox-based
149  *                                                 |            |
150  *                                           other ITRF     other ITRF
151  *                                                      w/o tidal effects
152  * </pre>
153  * <p>
154  * This is a utility class, so its constructor is private.
155  * </p>
156  * @author Guylaine Prat
157  * @author Luc Maisonobe
158  * @author Pascal Parraud
159  */

366     /** Get Earth Orientation Parameters history.
367      * <p>
368      * If no {@link EOPHistoryLoader} has been added by calling {@link
369      * #addEOPHistoryLoader(IERSConventions, EOPHistoryLoader) addEOPHistoryLoader}
370      * or if {@link #clearEOPHistoryLoaders() clearEOPHistoryLoaders} has been
371      * called afterwards, the {@link #addDefaultEOP1980HistoryLoaders(String, String,
372      * String, String, String)} and {@link #addDefaultEOP2000HistoryLoaders(String,
373      * String, String, String, String)} methods will be called automatically with
374      * supported file names parameters all set to null, in order to get the default
375      * loaders configuration.
376      * </p>
377      * @param conventions conventions for which EOP history is requested
378      * @param simpleEOP if true, tidal effects are ignored when interpolating EOP
379      * @return Earth Orientation Parameters history
380      * @exception OrekitException if the data cannot be loaded
381      */
382     public static EOPHistory getEOPHistory(final IERSConventions conventions, final boolean simpleEOP)
383         throws OrekitException {
384 
385         synchronized (EOP_HISTORY_LOADERS) {
386 
387             if (EOP_HISTORY_LOADERS.isEmpty()) {
388                 // set up using default loaders
389                 addDefaultEOP2000HistoryLoaders(null, null, null, null, null);
390                 addDefaultEOP1980HistoryLoaders(null, null, null, null, null);
391             }
392 
393             // TimeStamped based set needed to remove duplicates
394             OrekitException pendingException = null;
395             final SortedSet<EOPEntry> data = new TreeSet<EOPEntry>(new ChronologicalComparator());
396 
397             // try to load canonical data if available
398             if (EOP_HISTORY_LOADERS.containsKey(conventions)) {
399                 for (final EOPHistoryLoader loader : EOP_HISTORY_LOADERS.get(conventions)) {
400                     try {
401                         loader.fillHistory(conventions.getNutationCorrectionConverter(), data);
402                     } catch (OrekitException oe) {
403                         pendingException = oe;
404                     }
405                 }
406             }
407 
408             if (data.isEmpty() && pendingException != null) {
409                 throw pendingException;
410             }
411 
412             final EOPHistory history = new EOPHistory(conventions, data, simpleEOP);
413             history.checkEOPContinuity(EOP_CONTINUITY_THRESHOLD);
414             return history;
415 
416         }
417 
418     }

1200     /** Get the transform between two frames, suppressing all interpolation.
1201      * <p>
1202      * This method is similar to {@link Frame#getTransformTo(Frame, AbsoluteDate)}
1203      * except it removes the performance enhancing interpolation features that are
1204      * added by the {@link FramesFactory factory} to some frames, in order to focus
1205      * on accuracy. The interpolation features are intended to save processing time
1206      * by avoiding doing some lengthy computation like nutation evaluation at each
1207      * time step and caching some results. This method can be used to avoid this,
1208      * when very high accuracy is desired, or for testing purposes. It should be
1209      * used with care, as doing the full computation is <em>really</em> costly for
1210      * some frames.
1211      * </p>
1212      * @param from frame from which transformation starts
1213      * @param to frame to which transformation ends
1214      * @param date date of the transform
1215      * @return transform between the two frames, avoiding interpolation
1216      * @throws OrekitException if transform cannot be computed at this date
1217      */
1218     public static Transform getNonInterpolatingTransform(final Frame from, final Frame to,
1219                                                          final AbsoluteDate date)
1220         throws OrekitException {
1221 
1222         // common ancestor to both frames in the frames tree
1223         Frame currentF = from.getDepth() > to.getDepth() ? from.getAncestor(from.getDepth() - to.getDepth()) : from;
1224         Frame currentT = from.getDepth() > to.getDepth() ? to : to.getAncestor(to.getDepth() - from.getDepth());
1225         while (currentF != currentT) {
1226             currentF = currentF.getParent();
1227             currentT = currentT.getParent();
1228         }
1229         final Frame common = currentF;
1230 
1231         // transform from common to origin
1232         Transform commonToOrigin = Transform.IDENTITY;
1233         for (Frame frame = from; frame != common; frame = frame.getParent()) {
1234             commonToOrigin = new Transform(date,
1235                                              peel(frame.getTransformProvider()).getTransform(date),
1236                                              commonToOrigin);
1237         }
1238 
1239         // transform from destination up to common
1240         Transform commonToDestination = Transform.IDENTITY;
1241         for (Frame frame = to; frame != common; frame = frame.getParent()) {
1242             commonToDestination = new Transform(date,
1243                                                 peel(frame.getTransformProvider()).getTransform(date),
1244                                                 commonToDestination);
1245         }
1246 
1247         // transform from origin to destination via common
1248         return new Transform(date, commonToOrigin.getInverse(), commonToDestination);
1249 
1250     }

1252     /** Peel interpolation and shifting from a transform provider.
1253      * @param provider transform provider to peel
1254      * @return peeled transform provider
1255      * @exception OrekitException if EOP cannot be retrieved
1256      */
1257     private static TransformProvider peel(final TransformProvider provider)
1258         throws OrekitException {
1259 
1260         TransformProvider peeled = provider;
1261 
1262         boolean peeling = true;
1263         while (peeling) {
1264             if (peeled instanceof InterpolatingTransformProvider) {
1265                 peeled = ((InterpolatingTransformProvider) peeled).getRawProvider();
1266             } else if (peeled instanceof ShiftingTransformProvider) {
1267                 peeled = ((ShiftingTransformProvider) peeled).getRawProvider();
1268             } else if (peeled instanceof EOPBasedTransformProvider &&
1269                        ((EOPBasedTransformProvider) peeled).getEOPHistory().usesInterpolation()) {
1270                 peeled = ((EOPBasedTransformProvider) peeled).getNonInterpolatingProvider();
1271             } else {
1272                 peeling = false;
1273             }
1274         }
1275 
1276         return peeled;
1277 
1278     }
